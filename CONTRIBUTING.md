# Contributing to DWM - Modern & Hardened

First off, thank you for considering contributing to this fork! It's people like you that make the open-source community such a great place.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Describe the bug in detail and provide steps to reproduce.
- Mention your hardware (GPU/Monitor setup) if relevant to display issues.

### Suggesting Enhancements
- Open an issue with the "feature request" tag.
- Explain why this enhancement would be useful to the majority of users.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. Ensure the documentation is updated if you change functionality.
4. Issue that pull request!

## Style Guide
- **C Code:** Follow the standard suckless coding style (tab indentation, minimal dependencies).
- **Shell Scripts:** Use ShellCheck to ensure script robustness.
- **TOML:** Keep keys lowercase and organized by category.
